# 1차 프로젝트 기획 설계서

학번: 20602  
이름: 금재준  
프로젝트명: 나의 MBTI가 궁금해!

---

## 1. 프로젝트 개요

### 1) 주제

20문항으로 알아보는 나의 MBTI

### 2) 목적

Flask 웹 애플리케이션을 이용하여 사용자가 로그인한 뒤 MBTI 성격유형 검사를 진행하고, 자신의 성격 유형과 성향별 강도를 간단하게 확인할 수 있도록 한다.

서비스 대상은 자신의 성격 유형을 쉽고 빠르게 알아보고 싶은 학생 및 일반 사용자이다.

### 3) 기대 효과

- 간단한 로그인 기능을 통해 사용자별 접속 상태를 관리할 수 있다.
- 20개의 질문을 두 페이지로 나누어 부담 없이 검사할 수 있다.
- 검사 결과를 MBTI 유형, 유형별 설명, 성향별 그래프로 확인할 수 있다.
- Flask 라우팅, 폼 처리, 세션, Jinja2 반복문과 템플릿 상속을 학습할 수 있다.

## 2. UI 설계

### 1) 공통 화면(base.html)

- 모든 페이지가 상속하는 공통 템플릿이다.
- HTML 기본 구조, 페이지 제목, CSS 파일 연결을 담당한다.
- 로그인 상태이면 우측 상단에 세션에 저장된 사용자 아이디와 로그아웃 버튼을 표시한다.
- 각 페이지는 `title`, `body_class`, `content` 블록에 필요한 내용을 작성한다.

### 2) 로그인 화면(login.html)

- 아이디와 비밀번호를 입력하는 폼을 제공한다.
- 로그인에 실패하면 입력 폼 아래에 오류 메시지를 출력한다.
- 로그인에 성공하면 세션에 사용자 아이디를 저장하고 검사 안내 화면으로 이동한다.

### 3) 검사 안내 화면(main.html)

- MBTI 검사에 대한 설명, 총 문항 수, 검사 방법을 안내한다.
- 검사 시작 버튼을 누르면 1~10번 문항 화면으로 이동한다.
- 우측 상단에서 로그인 사용자와 로그아웃 버튼을 확인할 수 있다.

### 4) 검사 문항 화면(test.html)

- 하나의 HTML 템플릿을 1페이지와 2페이지에서 공통으로 사용한다.
- 1페이지에는 1~10번, 2페이지에는 11~20번 문항을 출력한다.
- Jinja2 반복문을 사용하여 질문과 선택지를 출력한다.
- 각 문항은 매우 그렇다, 그렇다, 보통이다, 아니다, 전혀 아니다 중 하나를 선택한다.
- 모든 문항의 라디오 버튼에 `required`를 적용하여 답변을 완료해야 다음 단계로 이동할 수 있다.
- 1페이지 답변은 2페이지의 hidden input으로 전달하여 최종 결과 계산에 함께 사용한다.

### 5) 결과 화면(result.html)

- 계산된 MBTI 유형을 큰 글씨로 출력한다.
- 유형별 별명과 설명을 출력한다.
- 에너지 방향, 인식 방식, 판단 기준, 생활 양식, 정체성의 성향 강도를 막대그래프로 표시한다.
- 다시 검사하기 버튼을 누르면 검사 안내 화면으로 이동한다.

## 3. 데이터 설계

### 1) 로그인 데이터

| 변수명 | 자료형 | 값 | 용도 |
|---|---|---|---|
| USER_ID | 문자열(str) | guest | 테스트용 아이디 |
| USER_PW | 문자열(str) | 1234 | 테스트용 비밀번호 |

### 2) 성향 기준 데이터(DIMENSIONS)

- 전체 자료형: 리스트(list)
- 각 항목의 자료형: 튜플(tuple)
- 형태: `list[tuple[str, str, str, str]]`

| 변수명 | 양수 성향 | 음수 성향 | 화면 표시명 |
|---|---|---|---|
| energy | E | I | 에너지 방향 |
| perception | S | N | 인식 방식 |
| judgement | T | F | 판단 기준 |
| lifestyle | J | P | 생활 양식 |
| identity | A | T | 정체성 |

### 3) 선택지 데이터(OPTIONS)

- 전체 자료형: 리스트(list)
- 각 선택지의 자료형: 딕셔너리(dict)
- `value`: 정수(int)
- `cls`: 문자열(str)

| 선택지 | 점수 | CSS 클래스 |
|---|---:|---|
| 매우 그렇다 | +3 | a2 |
| 그렇다 | +2 | a1 |
| 보통이다 | +1 | n |
| 아니다 | -1 | d1 |
| 전혀 아니다 | -2 | d2 |

### 4) 질문 데이터(QUESTIONS)

- 전체 자료형: 리스트(list)
- 각 질문의 자료형: 딕셔너리(dict)
- 각 딕셔너리는 `name`과 `text`를 문자열로 저장한다.
- 총 20문항이며 5개 성향 기준마다 4문항씩 배치한다.
- `name`은 `energy1`, `perception1`처럼 성향명과 번호를 결합한다.

### 5) 결과 설명 데이터(TYPE_INFO)

- 전체 자료형: 중첩 딕셔너리(dict)
- ISTJ, ENFP, ENTJ 등 16가지 MBTI 유형을 키로 사용한다.
- 각 유형은 별명 `nick`과 설명 `desc`를 문자열로 저장한다.
- 형태: `dict[str, dict[str, str]]`

### 6) 세션 데이터

| 세션 키 | 자료형 | 저장 값 | 용도 |
|---|---|---|---|
| user_id | 문자열(str) | guest | 로그인 상태 유지 및 사용자 표시 |

| 9 | 결과 출력 | POST | Jinja2, CSS | 유형 설명과 성향별 막대그래프 출력 |
| 10 | 템플릿 상속 | 전체 페이지 | Jinja2 extends, block | 공통 HTML 구조와 로그아웃 UI 재사용 |

## 5. URL 라우팅 설계

| URL 경로 | HTTP 메서드 | 연결 함수 | 설명 |
|---|---|---|---|
| `/` | GET | root() | 로그인 여부에 따라 `/login` 또는 `/main`으로 이동 |
| `/login` | GET, POST | login() | 로그인 화면 출력 및 계정 검증, 세션 저장 |
| `/logout` | GET | logout() | 세션 삭제 후 `/login`으로 이동 |
| `/main` | GET | main() | 검사 안내 페이지 출력 |
| `/test` | GET | test() | 1~10번 문항 출력 |
| `/test2` | GET, POST | test() | POST 시 11~20번 문항 출력, 직접 GET 접근 시 `/test`로 이동 |
| `/result` | POST | result() | 20개 답변을 계산하여 결과 페이지 출력 |

URL은 총 7개이며, `/test`와 `/test2`는 중복 코드를 줄이기 위해 하나의 `test()` 함수와 하나의 `test.html` 템플릿을 함께 사용한다.

## 6. 템플릿(HTML) 구조 설계

| 파일명 | 상속 페이지 | 역할 및 포함 내용 |
|---|---|---|
| base.html | 없음(최상위 부모) | 공통 HTML 구조, CSS 연결, 세션 사용자 표시, 로그아웃 버튼, block 정의 |
| login.html | base.html | 로그인 폼, 아이디·비밀번호 입력, 오류 메시지 |

| result.html | base.html | MBTI 결과, 유형 설명, 성향별 그래프, 다시 검사하기 버튼 |

## 7. 세션 설계

### 1) 세션 생성

- 로그인 성공 시 `session["user_id"] = USER_ID`로 사용자 아이디를 저장한다.
- Flask 세션 사용을 위해 `app.secret_key`를 설정한다.

### 2) 세션값 사용

- `base.html`에서 `session.user_id`를 읽어 우측 상단에 로그인 사용자를 표시한다.
- `/` 접속 시 세션 존재 여부에 따라 `/login` 또는 `/main`으로 이동한다.

### 3) 접근 제한

- `@app.before_request`로 모든 요청 전에 로그인 상태를 확인한다.
- `root`, `login`, `static` 이외의 페이지는 세션에 `user_id`가 있어야 접근할 수 있다.
- 비로그인 사용자가 검사 또는 결과 URL에 접근하면 `/login`으로 이동한다.

### 4) 로그아웃

- `/logout`에서 `session.clear()`를 실행하여 세션값을 삭제한다.
- 로그아웃 완료 후 `/login`으로 이동한다.

## 8. 검사 및 결과 처리 설계

### 1) 검사 진행

1. `/test`에서 `QUESTIONS[0:10]`을 출력한다.
2. 사용자가 1~10번에 답하면 `/test2`로 POST 요청을 전송한다.
3. 서버는 1페이지의 10개 답변이 모두 존재하는지 확인한다.
4. `/test2`에서 `QUESTIONS[10:20]`을 출력한다.
5. 1페이지 답변은 hidden input으로 2페이지 폼에 포함한다.
6. 전체 20개 답변을 `/result`로 POST 전송한다.

### 2) 점수 계산

- 각 성향은 4개 문항의 점수를 합산한다.
- 성향별 점수 범위는 -8점부터 12점까지이다.
- 합계가 0 이상이면 앞 글자(E, S, T, J, A)를 선택한다.
- 합계가 음수이면 뒤 글자(I, N, F, P, T)를 선택한다.
- 앞의 네 글자와 정체성 글자를 조합하여 `ESTJ-A`, `INFP-T` 형태로 출력한다.

### 3) 성향 강도 계산

- 기본 백분율 계산식: `round((score + 8) / 20 * 100)`
- 양수 성향이면 계산된 백분율을 사용한다.
- 음수 성향이면 `100 - percent`를 사용한다.
- 계산한 값은 결과 화면의 막대그래프 너비로 사용한다.

## 11. 실행 방법

1. 명령 프롬프트에서 프로젝트 폴더로 이동한다.
2. `pip install -r requirements.txt`를 실행한다.
3. `python app.py`를 실행한다.
4. 웹 브라우저에서 `http://localhost:5000`에 접속한다.
5. 아이디 `guest`, 비밀번호 `1234`로 로그인한다.
